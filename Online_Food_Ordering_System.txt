Online Food Ordering System – System Design Document

 Introduction
The Online Food Ordering System is a full‑stack web application designed to connect students with local food vendors. The platform allows students to browse menus, add items to a cart, place orders, and track delivery, while vendors manage food availability and orders. The system focuses on scalability, security, and smooth order workflow.

Tech Stack
Frontend: React.js
Backend: Spring Boot (Java)
Database: MySQL
Authentication: JWT‑based authentication

System Requirements
Functional Requirements
For Students
Register and log in securely
Browse food vendors and menus
Add/remove items from cart
Place orders
Track order status
For Vendors
Login and manage menu items
Toggle food availability (ON/OFF)
Accept or reject orders
Update order status
For Delivery Panel
View assigned orders
Update delivery status (Picked, On the Way, Delivered)

Non‑Functional Requirements
Secure API endpoints
Fast response time (< 2s per request)
Scalable architecture
Data consistency and integrity
Role‑based access control

System Architecture Overview (High Level Design)

Client‑Server Architecture

High Level Flow:

Backend Design (Low Level Design)

Responsibilities:
User login & registration
JWT token generation
Role‑based access
APIs:
POST /api/auth/login
POST /api/auth/register

Module 5: User & Vendor Management (Backend)
Responsibilities:
Manage users and vendors
Fetch user profiles
APIs:
GET /api/users/{id}
GET /api/vendors

Module 6: Menu Management (Backend)
Responsibilities:
Add/update/delete menu items
Toggle item availability
APIs:
POST /api/menu
PUT /api/menu/{id}/toggle
GET /api/menu/vendor/{vendorId}

Module 7: Order Management (Backend)
Responsibilities:

Manage order workflow

Preparing
Out for Delivery
Delivered
APIs:

Module 8: Delivery Management (Backend)
Responsibilities:
Assign delivery personnel

APIs:

Database Design
Main Tables
users (id, name, email, password, role)
vendors (id, name, availability)
menu_items (id, vendor_id, name, price, available)
orders (id, user_id, vendor_id, status, total_price)
order_items (id, order_id, menu_item_id, quantity)
delivery (id, order_id, delivery_status)

 Security Design
JWT‑based authentication
Password hashing using BCrypt
Role‑based endpoint protection
HTTPS for secure communication

Conclusion
This system design provides a scalable, secure, and modular Online Food Ordering System suitable for student use. The clear separation of frontend and backend modules ensures maintainability and future enhancements such as payment integration or mobile app support.
